# 中国大陆 App Store 特殊要求

> 覆盖 ICP 备案、网络安全法/PIPL 合规、游戏版号、年龄分级差异、AI 品牌词限制、实名认证及支付合规。
> 本文件仅列出中国大陆地区与全球通用规则不同或额外的要求。

---

## 一、ICP 备案（强制）

### 1.1 基本要求

自 2023 年 9 月起，Apple 要求所有中国大陆地区 App Store 上架的 App **必须持有有效 ICP 备案号**（非 ICP 经营许可证）。

| 项目 | 说明 |
|------|------|
| 适用范围 | 中国大陆区 App Store 上所有 App（含已上架存量 App） |
| 备案类型 | App 备案（非网站备案），通过工信部 App 备案系统 |
| 填写位置 | App Store Connect → App Information → 中国大陆区域信息 |
| 必填信息 | ICP 备案号（格式：京ICP备XXXXXXXX号-XA） |

### 1.2 获取 ICP 备案

1. 通过境内服务器托管商（阿里云/腾讯云/华为云等）的备案系统提交
2. 需要：域名实名认证 + 企业营业执照（或个人身份证）+ 法人身份证
3. 审核周期：通常 7-20 个工作日
4. ⚠️ 经验规则：外国公司/个人开发者需通过中国境内合作方申请

### 1.3 没有 ICP 备案的后果

- 新 App：无法提交审核
- 已上架 App：无法提交更新（已有版本不受影响但有下架风险）
- ⚠️ 经验规则：部分品类（教育、金融）审核更严格

## 二、网络安全法 / PIPL 合规

### 2.1 个人信息保护法（PIPL）

中国《个人信息保护法》（2021年11月1日生效）对 App 有以下要求：

| 要求 | 说明 | App Store 审核关联 |
|------|------|-----------------|
| 明示同意 | 收集个人信息前需明确告知并获得同意 | 5.1.1 |
| 最小必要 | 不得过度收集与服务无关的个人信息 | 5.1.1 / 5.1.2 |
| 隐私政策 | 必须有中文隐私政策，且内容符合 PIPL | 5.1.1 |
| 跨境传输 | 个人信息出境需通过安全评估或取得认证 | 5.1.1 |
| 删除权 | 用户有权要求删除个人信息 | 5.1.1(v) |

### 2.2 App 隐私合规自查（工信部 / 网信办要求）

中国监管机构定期通报 App 违规收集个人信息，被通报的 App 可能被 Apple 配合下架：

- **禁止**：首次启动未经同意即收集 IDFA / IMEI / MAC 地址
- **禁止**：用户拒绝权限后反复弹窗索要
- **禁止**：不提供个人信息删除功能
- **必须**：提供「撤回同意」操作入口
- **必须**：SDK 收集行为在隐私政策中逐一列明

### 2.3 数据出境安全评估

如果 App 将中国用户数据传输到境外服务器：

| 情形 | 是否需要安全评估 |
|------|----------------|
| 处理 100 万+ 用户数据出境 | ✅ 需要 |
| 上年度向境外提供 10 万+ 个人信息 | ✅ 需要 |
| 上年度向境外提供 1 万+ 敏感个人信息 | ✅ 需要 |
| 小规模、非敏感数据出境 | 可通过标准合同或认证 |

## 三、游戏版号

### 3.1 要求

| 类型 | 是否需要版号 |
|------|------------|
| 付费游戏 | ✅ 必须 |
| 含 IAP 的免费游戏 | ✅ 必须 |
| 完全免费无 IAP 游戏 | ⚠️ 经验规则：理论上需要，但执行力度不一 |
| 非游戏 App 含游戏化元素 | 通常不需要（除非核心功能是游戏） |

### 3.2 版号（ISBN）申请

- 由国家新闻出版署审批
- 通常需要通过有资质的中国出版商代理申请
- 审批周期：数月到一年不等
- 版号信息需在 App 内和 ASC 元数据中展示

### 3.3 Apple 执行情况

- Apple 会要求游戏类 App 在 ASC 中填写版号
- 未提供版号的游戏 App 可能无法上架或更新
- ⚠️ 经验规则：个别小体量游戏可能暂时未被强制，但风险持续存在

## 四、年龄分级差异

中国大陆的年龄分级系统与国际版有差异：

| 差异项 | 说明 |
|-------|------|
| 分级体系 | 中国区使用与国际区略有不同的分级标准 |
| 赌博内容 | 任何赌博相关内容（包括模拟赌博）在中国区管控更严格 |
| 暴力内容 | 血腥暴力内容标准比国际区更低 |
| 未成年保护 | 游戏类 App 需遵守未成年人防沉迷系统要求 |

### 4.1 未成年人防沉迷（游戏类）

如果 App 是游戏且面向中国用户：

- 必须接入实名认证系统
- 未成年人（未满 18 周岁）：仅限周五、六、日及法定节假日 20:00-21:00 游玩（国家新闻出版署 2021 年规定）
- 需提供防沉迷提示
- 未成年人消费限额（月度/单次）

## 五、AI 品牌词限制

⚠️ **以下为经验规则，非 Apple 明文。基于 2.3.7 条款推断和中国区审核实践。**

### 5.1 已观察到的限制

| 限制 | 说明 |
|------|------|
| App Name 不可含「ChatGPT」「GPT」等 | 2.3.7 竞品名 + 中国区 AI 监管 |
| 不可暗示使用未在中国备案的 AI 模型 | 中国 AI 生成内容管理规定 |
| 描述中不可宣称使用 OpenAI/Anthropic 等境外 AI 服务 | 中国区审核实践 |
| 「AI」一词本身可用，但需与实际功能匹配 | 2.3.1 |

### 5.2 安全做法

- 使用通用描述（如「智能助手」而非「GPT 驱动」）
- 如使用境外 AI API，确保数据不出境或已完成安全评估
- ⚠️ 中国区上架 AI 类 App 可能需要「算法备案」（视具体 AI 功能）

## 六、实名认证

### 6.1 开发者实名

- 中国区开发者账号需实名认证（个人：身份证 + 手机号；公司：营业执照 + 法人身份证）
- D-U-N-S 编号（公司账号）需关联中国大陆实体

### 6.2 用户实名（特定类别）

| App 类型 | 是否需要用户实名 |
|---------|----------------|
| 游戏 | ✅ 必须（防沉迷要求） |
| 社交/UGC | ✅ 需要（网络安全法 / App 信息服务管理规定） |
| 金融 | ✅ 必须（KYC 要求） |
| 视频/新闻 | ✅ 需要（广电总局要求，另需《信息网络传播视听节目许可证》或《互联网新闻信息服务许可证》） |
| VPN | ❌ 中国区 App Store 不允许上架 VPN App（见 `../by-app-type/vpn.md`） |
| 工具/效率 | 通常不需要 |

## 七、支付合规

### 7.1 IAP 与第三方支付

| 场景 | 要求 |
|------|------|
| 数字内容/服务 | **必须使用 IAP**，不能替代为微信/支付宝支付（3.1.1） |
| 实物商品 | **不可使用 IAP**，应使用微信/支付宝/银联等（3.1.3(e)） |
| 线下服务（打车/外卖） | **不可使用 IAP**，使用第三方支付（3.1.3(e)） |
| App 内电商（实物） | 可接入微信/支付宝，必须走第三方支付 |

### 7.2 注意事项

- 数字内容走微信/支付宝支付绕过 IAP 会被拒审（3.1.1），这是全球规则在中国区同样适用
- 实物商品接入微信/支付宝是**合规且必要的**（Apple 不提供中国境内的非 IAP 支付方案）
- ⚠️ 经验规则：部分审核员对「数字内容 vs 实物」的界定较严，模糊地带（如虚拟+实物混合）需在 Review Notes 中说明

### 7.3 中国区 App Store 佣金费率（2026-03-15 下调）

**2026-03-15 起，中国区 App Store 佣金下调**（中国市场专属，不适用其他地区）：

| 费率类型 | 旧费率 | 新费率（2026-03-15+） | 适用 |
|---------|--------|----------------------|------|
| 标准 IAP / 付费 App | 30% | **25%** | 未参加 Small Business Program 的开发者 |
| Small Business Program | 15% | **12%** | 年 proceeds < $1M USD 的开发者 |
| Mini Apps（小程序类） | 15% | **12%** | 通过 4.7 条款分发的 mini app / HTML5 容器 |

**适用范围**：
- 仅中国大陆 App Store（CN storefront）的 IAP 交易
- 订阅类（auto-renewable）：第 2 年起的 loyalty rate **由 15% 下调至 12%** ✅（Apple 官方明文）
- 其他地区的 storefront（US / EU / JP 等）**费率不变**

**背景**：Apple 于 [2026-03-12 Developer News](https://developer.apple.com/news/?id=dadukodv) 正式公告《Adjustments to the China storefront of the App Store on iOS and iPadOS》，称系与中国监管机构讨论后的调整，**无需签署更新条款即可享受新费率**。生效日 2026-03-15。

**对中国区开发者的影响**：
- [ ] 内部 LTV / 单价 / ROI 计算模型需按新费率重算
- [ ] 订阅定价无需调整，净收入提升 5pp（30→25）或 3pp（15→12）
- [ ] 与 Apple 既有签约无需重新签署；费率在 ASC 后台和 Payments & Financial Reports 中自动切换
- [ ] ⚠️ 以 Apple 最新公告为准：本项费率调整仅针对中国区；不要错用到 US/EU 定价模型

**来源**：✅ Apple 官方公告
- [Apple Developer News 2026-03-12: Adjustments to the China storefront of the App Store on iOS and iPadOS](https://developer.apple.com/news/?id=dadukodv)
- 媒体解读：[Bloomberg](https://www.bloomberg.com/news/articles/2026-03-13/apple-lowers-app-store-cut-to-25-from-30-in-china-to-fend-off-local-regulators) / [TechCrunch](https://techcrunch.com/2026/03/13/apple-drops-commission-rates-in-china-25-percent/) / [The Register](https://www.theregister.com/2026/03/13/apple_china_appstore_commissions_fall/)

## 八、审核 Checklist（中国区专项）

### 提交前必检

- [ ] ICP 备案号已获取并填入 ASC
- [ ] 中文隐私政策 URL 有效、内容符合 PIPL
- [ ] 用户数据不出境，或已完成数据出境安全评估/标准合同
- [ ] 首次启动不在用户同意前收集设备标识符
- [ ] 提供个人信息删除和撤回同意功能
- [ ] 游戏类：已获取版号（ISBN）并在 App 内展示
- [ ] 游戏类：已接入实名认证和防沉迷系统
- [ ] **2.3.7**：App Name / Description 不含境外 AI 品牌词（⚠️ 经验规则）
- [ ] AI 类 App：检查是否需要算法备案（⚠️ 经验规则）
- [ ] 数字内容支付走 IAP，实物商品走第三方支付
- [ ] 元数据使用简体中文
