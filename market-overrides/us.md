# 美区 App Store 特殊要求

> 覆盖美区外部购买链接 Entitlement、NFT 外链、COPPA、CCPA、州级法律及相关费用结构。
> 本文件仅列出美国地区与全球通用规则不同或额外的要求。

---

## 一、外部购买链接（US 区）

### 1.1 概述

美区（美国）App 可在 App 内放置链接引导用户到外部网站购买数字内容。这基于 Apple 在全球范围的政策调整（受 Epic v. Apple 诉讼影响）。

| 项目 | 说明 |
|------|------|
| Entitlement | `StoreKit External Purchase Link Entitlement` |
| 申请要求 | 无需特殊审批，在 Developer Portal 申请即可 |
| 佣金 | ⚠️ 以 Apple 最新公告为准：2024 年法院裁决后佣金政策持续变动中。历史上为 27%（标准）或 12%（小型开发者 <$1M/年） |
| 报告义务 | 开发者须每月向 Apple 报告通过外部链接产生的交易金额 |

### 1.2 与 EU 区的差异

| 对比项 | 美区 | EU 区 |
|-------|------|-------|
| 佣金率 | 27% / 12% | ~7%（CTC + Acquisition） |
| 报告义务 | 必须每月报告 | 必须每月报告 |
| 系统披露页 | 是 | 是 |
| 可同时展示 IAP | 是（可在同一 App 中） | ⚠️ 待实测 |
| 替代 App 市场 | ❌ 不支持 | ✅ 支持 |

### 1.3 外部购买链接实现

```swift
import StoreKit

// 检查是否可使用外部购买链接
if ExternalPurchaseLink.canOpen {
    // 系统自动展示披露页
    try await ExternalPurchaseLink.open()
}
```

### 1.4 披露页内容

用户点击外部链接后，系统会展示一个披露页：
- 告知用户即将离开 App
- 说明 Apple 不负责外部交易的退款、隐私保护和安全
- 用户确认后跳转到外部 URL

### 1.5 报告流程

- 每月通过 App Store Connect 的外部购买报告功能提交
- 报告内容：通过外部链接产生的总交易额
- Apple 据此计算并收取佣金
- 逾期未报告可能导致 App 下架

## 二、NFT / 数字收藏品

### 2.1 当前政策

| 允许 | 禁止 |
|------|------|
| 在 App 内展示用户已有的 NFT | 通过非 IAP 渠道在 App 内购买 NFT（数字内容必须走 IAP） |
| 从 App 链接到外部 NFT 市场 | 在 App 内引导用户使用加密钱包购买 |
| 展示 NFT 元数据和图片 | 解锁 App 功能的 NFT 不通过 IAP（3.1.1） |
| 使用外部购买链接引导到 NFT 市场 | — |

### 2.2 NFT 外链允许（3.1.1 例外）

- App 可链接到外部网站查看/交易 NFT
- 此链接不需要通过 IAP
- 但如果 NFT 用于**解锁 App 内功能/内容**，则该功能/内容的价值必须通过 IAP 提供等价购买方式

## 三、COPPA（儿童在线隐私保护法）

### 3.1 适用条件

| 条件 | 说明 |
|------|------|
| 目标用户 <13 岁 | App 面向 13 岁以下儿童 |
| 实际用户 <13 岁 | App 实际收集 13 岁以下用户信息（即使不是目标用户） |
| Made for Kids | App Store 中标记为 Made for Kids 的 App |

### 3.2 COPPA 要求

| 要求 | 说明 | App Store 审核关联 |
|------|------|-----------------|
| 家长可验证同意 | 收集儿童信息前需获得家长可验证的同意 | 1.3 |
| 数据最小化 | 仅收集必要的儿童数据 | 5.1.1 |
| 不可定向广告 | 不可对 13 岁以下用户投放基于行为的广告 | 1.3 / 5.1.1 |
| 不可 tracking | 不可对儿童进行跨 App tracking | 1.3 |
| 隐私政策 | 须明确说明儿童数据收集和使用方式 | 5.1.1 |

### 3.3 与 App Store Kids Category 的关系

选择 Kids Category 的 App 自动受到更严格限制（见 `../by-app-type/kids.md`），且必须符合 COPPA。

## 四、CCPA（加州消费者隐私法）

### 4.1 适用条件

适用于满足以下任一条件的企业：

- 年收入 ≥ $25M
- 处理 ≥ 100,000 加州消费者数据
- 50%+ 收入来自出售消费者数据

### 4.2 核心要求

| 要求 | 说明 |
|------|------|
| 知情权 | 告知用户收集了什么数据 |
| 删除权 | 应用户要求删除数据（与 5.1.1(v) 一致） |
| 退出权 | 提供 "Do Not Sell My Personal Information" 选项 |
| 非歧视 | 不可因用户行使权利而降低服务质量 |

### 4.3 CPRA 更新（2023年生效）

CPRA（加州隐私权法案）扩展了 CCPA：

| 新增 | 说明 |
|------|------|
| 纠正权 | 用户可要求纠正不准确信息 |
| 限制使用权 | 用户可限制敏感个人信息的使用 |
| 自动化决策 | 用户有权知悉和退出自动化决策 |

## 五、州级法律

### 5.1 加州 SB 362（Delete Act）

| 项目 | 说明 |
|------|------|
| 生效 | 2024 年 |
| 适用 | 数据经纪人（data brokers）|
| 要求 | 消费者可通过统一平台一键要求所有数据经纪人删除数据 |
| 对 App 的影响 | 如果 App 被认定为数据经纪人（向第三方出售数据），需在加州 AG 注册并遵守 |

### 5.2 其他州级隐私法

| 州 | 法律 | 生效 | 关键差异 |
|---|------|------|---------|
| 弗吉尼亚 | VCDPA | 2023 | 与 CCPA 类似，适用门槛不同 |
| 科罗拉多 | CPA | 2023 | 包含 Universal Opt-Out 机制 |
| 康涅狄格 | CTDPA | 2023 | 包含忠诚度计划豁免 |
| 犹他 | UCPA | 2023 | 较宽松，仅限 opt-out 权利 |
| 德克萨斯 | TDPSA | 2024 | 无收入/数据量门槛 |
| 蒙大拿 | MCDPA | 2024 | 适用于所有处理 MT 居民数据的企业 |

### 5.3 对 App 开发的实际影响

- **最佳实践**：按最严格的州法律（加州）设计隐私功能，一次合规覆盖所有州
- 在隐私政策中覆盖所有适用州的权利
- 提供统一的数据删除/导出入口

## 六、年龄验证（US 各州）

从 iOS 26.4 开始，Apple 在部分美国州启用系统级年龄验证：

| 地区 | 要求 | 生效日期 |
|----|------|---------|
| 澳大利亚、巴西、新加坡 | 18+ App 需年龄验证 | 2026-02-24 |
| 犹他 | 18+ App 需年龄验证 | 2026-05-06 |
| 路易斯安那 | 18+ App 需年龄验证 | 2026-07-01 |
| 更多州待跟进 | — | 持续更新 |

**对开发者的影响**：
- 年龄验证由系统处理，开发者无需额外实现
- 确保年龄分级准确即可
- 见 `../references/store-metadata-compliance.md` 的年龄分级章节

## 七、审核 Checklist（美区专项）

### 提交前必检

- [ ] **COPPA**（如面向 <13 岁用户）：已获取家长可验证同意
- [ ] **COPPA**：不对 <13 岁用户投放行为广告或 tracking
- [ ] **CCPA/CPRA**（如适用）：提供 "Do Not Sell My Personal Information" 选项
- [ ] **CCPA/CPRA**：提供数据删除/导出/纠正功能
- [ ] **外部购买链接**（如使用）：已申请 Entitlement
- [ ] **外部购买链接**：正确触发系统披露页
- [ ] **外部购买链接**：已配置月度报告流程（27% / 12% 佣金）
- [ ] **NFT**（如涉及）：NFT 功能/内容解锁有 IAP 等价购买方式
- [ ] 年龄分级准确（部分州启用系统级年龄验证）
- [ ] 隐私政策覆盖 CCPA + COPPA（如适用）条款

### 美区常见拒审场景

| 场景 | 原因 | 修复 |
|------|------|------|
| 儿童 App 包含行为广告 | COPPA + 1.3 | 移除行为广告，改用上下文广告 |
| 外部购买链接未触发披露页 | Apple 政策 | 使用 StoreKit ExternalPurchaseLink API |
| NFT 解锁功能不提供 IAP 选项 | 3.1.1 | 提供 IAP 等价购买方式 |
| 隐私政策无 CCPA 条款 | 5.1.1 | 补充加州消费者权利说明 |
| 未按月报告外部购买金额 | Apple 合约 | 及时提交报告 |
