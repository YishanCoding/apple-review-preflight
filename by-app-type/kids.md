# 儿童类 App 专项风险（Kids Category）

> 本文件仅记录超出通用审核规则的专项要求。适用于 App Store 年龄分级为"儿童"（5岁及以下 / 6–8岁 / 9–11岁）的 App，以及任何主要面向儿童用户的 App。

---

## 1. 隐私：COPPA & GDPR 儿童条款

**规则来源：App Store Review Guidelines 1.3 / 5.1.1**

- **COPPA（美国）**：App 面向13岁以下儿童，必须在收集任何个人信息前取得可核实的家长同意（Verifiable Parental Consent）。禁止收集儿童的精确地理位置（除非有明确家长同意）。
- **GDPR-K（欧盟）**：处理16岁以下儿童数据必须取得父母/监护人同意（各成员国可降至13岁，但需逐国核查）。
- **实操要求**：
  - 隐私政策必须包含专门的儿童隐私章节，说明收集哪些数据、如何使用、家长如何要求删除。
  - 如有用户注册流程，必须包含年龄验证环节，不能依赖用户自报年龄。
  - 不得要求儿童提供超过App功能所需的个人信息（姓名、邮箱、头像等均算）。

---

## 2. 禁止第三方数据分析和广告

**规则来源：App Store Review Guidelines 1.3(e) / 5.1.1(i)**

- **默认禁止**：行为广告、定向广告、跨 App 追踪（ATT 框架在儿童 App 中不可使用）。
- **默认禁止**：第三方数据分析 SDK（Firebase Analytics、Amplitude、Mixpanel、AppsFlyer、Adjust 等），即使不追踪个人身份信息。
- **例外（Guideline 1.3 允许）**：仅用于收集性能数据（崩溃报告、性能指标）的分析，且该分析提供商必须满足：
  - 不将数据用于广告目的
  - 不跨 App 关联数据
  - Apple 可接受的提供商：Xcode Crashlytics（需验证其儿童合规设置）、Firebase Crashlytics（需关闭 Analytics）
- **MMP（移动归因平台）**：Adjust / AppsFlyer / Branch 均不得集成，即使仅用于归因。
- **实操**：提交前必须用 Charles Proxy / mitmproxy 验证：App 运行期间无任何第三方域名请求（广告/分析相关）。

---

## 3. 禁止 App 外链接

**规则来源：App Store Review Guidelines 1.3(f)**

- 儿童 App 内不得包含任何指向 App 外的链接，包括：
  - 其他 App 的链接（含 App Store 页面）
  - 网站链接（含隐私政策页面的外部链接）
  - 社交媒体分享功能
  - "更多 App"入口
- **例外：家长专区（Parental Gate）保护的内容**（见下节）。
- **隐私政策链接**：必须放在家长专区后，或通过 App Store Connect 的隐私政策 URL 字段提供（不在 App 内直接跳转）。

---

## 4. 家长门槛（Parental Gate）实现要求

**规则来源：App Store Review Guidelines 1.3(f) / App Store Connect Help**

家长专区是解锁外链等成人内容的唯一合规路径。实现要求：

- **必须是认知挑战**，不能是简单点击确认。Apple 建议：数学题（如"1234 + 5678 = ?"）、输入特定随机数字等。
- **禁止使用**：
  - 固定密码或 PIN 码
  - 生物识别（Face ID / Touch ID）——儿童也会使用这些功能
  - 仅靠复选框"我是家长"
- **UI 规范**：
  - 家长专区界面必须明确告知家长"儿童通常无法完成以下步骤"
  - 通过后必须有明确的"现在离开 App"确认
- **必须保护的内容**：外部链接、购买入口（IAP 以外）、社区/聊天功能、其他 App 推荐。

---

## 5. 儿童 App 中允许的例外

| 功能 | 是否允许 | 条件 |
|------|----------|------|
| 崩溃报告 / 性能分析 | 允许 | 提供商不得将数据用于广告，不得跨 App 关联 |
| Apple 原生广告（SKAdNetwork） | 不允许 | 即使无个人识别信息 |
| 仅收集匿名聚合指标（如页面停留时长） | 灰色地带 | 强烈建议避免，Apple 审核员态度不一 |
| 家长专区后的 IAP 购买 | 允许 | 需经家长专区保护 |
| 家长专区后的外部链接 | 允许 | 需经家长专区保护 |
| 第一方用户数据收集（用于改善 App 体验） | 受限 | 必须满足 COPPA/GDPR-K 同意要求 |

---

## 6. 年龄分级问卷填写（儿童类别）

**在 App Store Connect → App 信息 → 年龄分级中**：

- 如 App 进入儿童专区，Primary Category 或 Secondary Category 必须选择 Kids。
- 问卷中所有内容类别（暴力、性内容、恐怖、赌博等）建议全部选"无"，有一项不符会导致无法进入儿童专区。
- 年龄段选择（5岁及以下 / 6–8岁 / 9–11岁）必须与 App 实际目标用户一致，Apple 会验证内容难度与界面设计。
- **2026-01-31 更新**：年龄分级问卷新增 13+/16+/18+ 选项，儿童 App 不受影响，但需确认不误选新选项。

## 常见拒审场景

1. **集成了 Firebase Analytics**（即使未显式调用，SDK 初始化时会发送数据）→ 完全移除或替换为仅崩溃报告模式。
2. **"更多游戏"按钮**直接跳转 App Store 未经家长专区保护。
3. **隐私政策链接**在 App 内可直接点击跳转外部浏览器。
4. **用户名/头像设置**要求输入真实姓名或上传照片，违反最小数据收集原则。
5. **推送通知**要求注册设备标识符，需确认推送提供商不将数据用于广告。
