# tvOS App 专项风险

> 本文件仅记录超出通用审核规则的专项要求。适用于 Apple TV 平台（tvOS）App。

---

## 1. 焦点导航（Focus Engine）要求

**规则来源：App Store Review Guidelines 2.1 / Human Interface Guidelines for tvOS**

tvOS 使用焦点引擎（Focus Engine）管理用户界面导航，所有交互元素必须支持焦点模型：

- **所有可交互元素必须可聚焦**：按钮、列表项、图片（如可点击）必须正确实现 `UIFocusItem` 协议，否则无法通过遥控器访问。
- **焦点移动必须合理**：上/下/左/右方向键移动焦点时，逻辑路径必须符合视觉布局，不允许焦点"跳跃"到不相关的元素。
- **焦点陷阱（Focus Trap）**：禁止出现焦点无法离开某区域的情况（如弹窗打开后无法用遥控器关闭）。
- **焦点记忆**：返回上级页面时，应恢复上次的焦点位置（系统默认行为，SwiftUI/UIKit 需注意自定义布局是否破坏此行为）。
- **视觉反馈**：聚焦状态必须有明显的视觉区分（系统默认提供放大 + 阴影效果，自定义 UI 必须实现等效效果）。
- **UIFocusGuide**：复杂布局（如网格、瀑布流）需要使用 `UIFocusGuide` 引导焦点移动路径，避免死角。

**审核测试方法**：审核员会用 Siri Remote 实际操控 App，焦点导航问题是 tvOS 最常见的拒审原因之一。

## 2. Siri Remote 交互规范

**规则来源：Human Interface Guidelines for tvOS / 审核实践**

Siri Remote（第二代及以后）的各按钮有规范用法，违反会导致拒审：

| 按钮/手势 | 标准用法 | 禁止用法 |
|---------|---------|---------|
| 点按触控板（Select） | 确认/选择当前焦点项 | 不能用于"返回" |
| 触控板轻扫（Swipe） | 移动焦点 / 滚动内容 | 不能覆盖用于触摸屏的复杂手势 |
| 菜单键（Back/Menu） | 返回上一级 / 按住返回主屏 | 不能禁用此键 |
| 播放/暂停键 | 媒体播放控制 | 不能用于非媒体操作 |
| 主屏键（TV 键） | 系统级，App 不可拦截 | 不能覆盖 |
| 音量键 | 系统级音量控制 | 不能覆盖 |

**触控板手势**：
- 触控板支持轻扫、点按，不支持多点触控。
- 不能依赖复杂触摸手势（如三指滑动），因为电视遥控器无此功能。

## 3. Top Shelf Extension 内容要求

**规则来源：App Store Review Guidelines / Human Interface Guidelines for tvOS**

Top Shelf Extension 是 Apple TV 主屏幕上的内容展示区域，限制如下：

- **内容必须与 App 直接相关**：展示的内容（图片、视频预览）必须是 App 内的实际内容，不能展示第三方内容或广告。
- **图片规格**：
  - 宽幅 Top Shelf：1920×720 px（静态图）
  - 分段式 Top Shelf：动态内容，图片比例 16:9，最小 548×274 px
- **内容质量**：图片必须高清，不能有压缩伪影或低分辨率放大。
- **不允许**：在 Top Shelf 展示促销/广告性质的内容（如"限时优惠"banner）。
- **点击行为**：Top Shelf 内容点击后必须打开 App 内对应的内容页面，不能只打开 App 首页。
- **内容刷新**：建议定期刷新 Top Shelf 内容，长期显示过时内容可能影响用户体验评分。

## 4. tvOS 截图要求

**规则来源：App Store Connect Help**

- **尺寸**：1920×1080 px（16:9 横向），不接受其他尺寸。
- **数量**：最少 1 张，最多 10 张。
- **格式**：PNG 或 JPEG，PNG 推荐（无损）。
- **内容要求**：
  - 必须展示 tvOS 上的实际 App 界面（不能用 iPhone/iPad 截图）。
  - 截图中不能包含 Apple TV 硬件边框（除非使用 Apple 官方设备模型）。
  - 不能包含价格信息或促销内容。
- **预览视频**：tvOS 支持 App 预览视频，分辨率必须是 1920×1080 或 3840×2160，时长 15–30 秒。
- **注意**：tvOS 截图与 iOS/macOS 截图完全独立，不能共用。

## 5. 无触控交互的 UI 设计合规

tvOS 没有触摸屏，UI 设计必须完全适配遥控器操作：

- **最小焦点元素尺寸**：可聚焦元素的点击区域至少 44×44 pt（同 iOS HIG），确保焦点视觉清晰。
- **禁止的 UI 模式**：
  - 需要精确点击的小按钮（如关闭弹窗的"×"，在 tvOS 上几乎无法操作）
  - 下拉菜单（需改为全屏选择界面或覆盖层）
  - 需要长按屏幕区域（遥控器长按 Select 键有特殊含义）
  - 拖拽排序（tvOS 不支持，需改为选中后用方向键移动的交互）
- **键盘输入**：tvOS 系统键盘操作繁琐，尽量减少文字输入需求，优先提供搜索建议、历史记录等减少输入的方案。
- **文字输入替代**：对于登录等必须输入的场景，建议支持 Sign in with Apple 或二维码扫码（手机辅助）方式。

## 6. 常见 tvOS 拒审场景

| 场景 | 原因 | 解决方案 |
|------|------|---------|
| 焦点无法到达部分按钮 | Focus Engine 配置错误 | 检查所有 UIButton/SwiftUI Button 的 focusable 属性 |
| 菜单键无法返回 | 拦截了系统事件 | 不要 override `pressesEnded(_:with:)` 拦截 Menu 键 |
| 截图是 iPhone 界面 | 截图平台错误 | 提供 1920×1080 tvOS 专用截图 |
| App 打开后无焦点状态 | 初始焦点未设置 | 确保首屏有默认焦点元素 |
| Top Shelf 点击后打开 App 首页 | 深度链接未实现 | 实现 `TVTopShelfProvider` 的 `openURL` 深度链接 |
| 弹出对话框后焦点消失 | 模态视图 Focus 处理错误 | 确保模态视图内有可聚焦元素 |
| App 仅为 iOS App 的缩放版 | 内容价值不足（2.1） | 至少针对大屏/遥控器优化核心功能 |
| 文字在 10 英尺距离下无法阅读 | 字号过小 | 遵循 tvOS HIG 字号规范（正文至少 29pt） |
