# E-commerce — 电商 / 实物商品类 App 专项

> 本文件只覆盖**电商 / 实物商品类 App 的增量风险**。
> 通用检查项见 `all-apps.md`。
> 美区外部支付规则见 `../market-overrides/us.md`。
> 欧盟 DMA/替代支付见 `../market-overrides/eu-eea.md`。

---

## 1. IAP 边界判定（3.1.1）

### 1.1 不需要 IAP 的情况

以下属于「实物商品/现实世界服务」，**不走 IAP，可走任意支付方式**（Stripe、微信、支付宝等）：

- 实物商品（衣服、食品、家具、电子产品）
- 现实世界服务（外卖、约车、家政、美容预约）
- 订阅实物配送（生鲜盒子、杂志实体版）
- 酒店预订、机票、票务

### 1.2 必须走 IAP 的情况

- 数字内容（电子书、数字音乐、App 内虚拟物品）
- 数字功能解锁（去广告、高级功能、皮肤）
- App 内订阅（数字服务）

### 1.3 混合模式的判定（最复杂区域）

| 场景 | IAP 要求 | 说明 |
|------|---------|------|
| 实物商品 + 送货跟踪功能（免费） | 不需要 | 功能服务于实物购买 |
| 实物商品 + 高级会员（数字特权，如免运费） | **需要 IAP** | 数字特权必须走 IAP |
| 实物商品 App + AR 试穿功能（免费） | 不需要 | AR 是辅助工具 |
| 实物商品 App + AI 穿搭建议（付费） | **需要 IAP** | 数字服务付费 |
| 订阅「无限退换货」（与实物绑定） | 灰区 | 建议咨询 Apple，倾向不需要 |

**判断原则**：付费解锁的是数字功能/内容 → 走 IAP；付费购买/使用实物或现实服务 → 不走 IAP。

## 2. 游客浏览（5.1.1(v)）

- [ ] 商品列表页：无需登录即可浏览
- [ ] 商品详情页（包括价格、图片、描述）：无需登录
- [ ] 搜索功能：无需登录
- [ ] 只有以下操作才触发登录：加入购物车、结账、查看订单历史、收藏商品

**为什么重要**：电商 App 是 5.1.1(v) 「强制注册才能浏览」的重灾区。审核员打开 App 第一件事是不登录点「查看商品」，如果被跳转到注册页，直接拒审。

## 3. 履约信息透明（2.3.1 / FTC 规则）

### 3.1 必须在购买路径上可见的信息

- [ ] 预计发货时效（「3-5 个工作日发货」）
- [ ] 退换货政策（退货窗口期、退款流程、费用承担）
- [ ] 运费计算（结账前可见，不能在最后一步才显示运费）

**具体位置要求**：
- 商品详情页：发货时效
- 购物车/结账页：退换货政策链接、运费明细
- 订单确认页：完整订单信息含退换货说明

### 3.2 不得包含的误导性内容

- [ ] 不虚标「原价」制造虚假折扣（「原价 ¥999，现价 ¥99」但实际从未以 ¥999 销售）
- [ ] 不用「限时特价」但无具体截止时间（2.3.1 元数据准确性同样适用于 App 内文案）

## 4. 3D 商品预览的文案边界（3.1.1 随机虚拟物品披露）

适用于：AR 试穿、3D 商品展示、虚拟试鞋等功能。

**允许的表达**：
- 「AR 试穿」「3D 预览」「虚拟试戴」「360° 展示」

**禁止的表达**（会被认定为 gambling 机制）：
- 「开盲盒」「随机搭配」「幸运试穿」（如果用户不知道会得到什么）
- 任何暗示「随机」「抽取」「惊喜」的表达，如果背后是确定的商品展示

**为什么重要**：3D 展示本身没问题，但文案游戏化（包装成抽奖）会触发 3.1.1（随机虚拟物品 odds 披露）或赌博条款审查。

## 5. 账号删除时订单历史的保留边界（5.1.1(v)）

Apple 要求账号删除功能「真实生效」，但电商有合法的数据保留需求：

### 5.1 允许保留的数据（符合法律要求）

- 订单记录：税务合规通常要求保留 5-7 年（各地区不同）
- 发票/收据：同税务保留要求
- 退换货记录：履约纠纷证明

### 5.2 必须删除的数据（账号删除时）

- 个人身份信息（姓名、手机号、邮箱、地址）
- 支付信息（银行卡、Apple Pay 绑定）
- 浏览历史、收藏、购物车
- 营销偏好和画像数据

### 5.3 合规实践

- [ ] 账号删除时说明数据保留政策：「您的订单记录将在税务要求期满（7年）后删除，个人信息将立即删除」
- [ ] 删除个人信息与保留必要履约记录（匿名化处理）
- [ ] 在隐私政策中说明数据保留时长和依据

**为什么重要**：直接拒绝用户删除账号（「已有订单，无法删除」）是 5.1.1(v) 拒审。正确做法是删除账号同时保留匿名化订单记录。

## 6. 外部支付链接规则

### 6.1 美区（Epic vs Apple 判决后）

- [ ] 美区 App 可在符合 Apple 外部购买链接规则的情况下展示外部购买链接；是否有报告或费用义务以 Apple 当前条款为准
- [ ] 外部支付链接必须使用 Apple 规定的标准文案和样式（Entitlement 申请后生效）
- [ ] 外链跳转前必须展示 Apple 的标准「即将离开 App Store」警告
- [ ] 详细规则见 `../market-overrides/us.md`

### 6.2 其他地区

- [ ] 在非美区（包括中国区）App 内展示外部支付链接（用于解锁数字内容）属于 3.1.1 违规
- [ ] 实物商品支付不受此限制（可以直接在 App 内接入任何支付 SDK）

## 7. Sign in with Apple 对电商 App 的要求（4.8）

电商 App 通常有多种登录方式，以下规则适用：

- [ ] 如果支持「微信登录」「Google 登录」「Facebook 登录」等第三方 OAuth，**必须同时提供** Sign in with Apple
- [ ] 如果只有手机号+验证码登录（非第三方 OAuth）：不强制要求 Sign in with Apple
- [ ] Apple 登录后用户的邮箱可能是 Apple 中继邮箱（`xxx@privaterelay.appleid.com`），营销系统需支持这类邮箱格式

**为什么重要**：4.8 硬规则。许多电商 App 接入了微信或 Google 登录但忘记加 Apple 登录，这是高频拒审原因。

## 8. 常见被拒场景

| 场景 | 被拒条款 | 修复 |
|------|---------|------|
| 商品列表需要登录才能查看 | 5.1.1(v) | 游客可浏览，仅购买时登录 |
| 结账最后一步才显示运费 | 2.3.1 / FTC | 购物车页面显示运费估算 |
| 账号删除按钮显示「有未完成订单，无法删除」 | 5.1.1(v) | 允许删除账号，保留匿名化订单 |
| 微信登录无 Sign in with Apple | 4.8 | 添加 Apple 登录选项 |
| 数字会员费用走外部支付（非美区） | 3.1.1 | 数字特权改走 IAP |
| 3D 试穿功能被描述为「开盲盒」 | 3.1.1 | 文案改为「AR 试穿」 |
