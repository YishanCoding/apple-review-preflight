# UGC / Social — UGC 社区类 App 专项

> 本文件只覆盖**UGC / 社区类 App 的增量风险**。
> 通用检查项见 `all-apps.md`。

---

## 1. 四件套：缺一必拒（1.2 条款）

Apple 对 UGC App 有明确的四件套要求，**全部缺失任何一项均会拒审**：

### 1.1 内容过滤机制

- [ ] 用户发布内容前有过滤层（关键词过滤、图片审核、视频审核）
- [ ] 支持 AI 辅助过滤 + 人工复审的双重机制（仅关键词过滤在内容丰富的社区中容易被认为不充分）
- [ ] 新用户发布内容可配置为「待审」状态后再公开

**为什么重要**：Apple 审核员会测试发布违规内容是否会立即公开展示。

### 1.2 举报机制

- [ ] 每条用户内容（帖子/评论/消息/视频）旁边有举报入口
- [ ] 举报类别完整：垃圾内容、骚扰、仇恨言论、裸露/色情、自残/自杀、版权侵犯
- [ ] 举报后有明确反馈（「已收到举报，我们将在 24 小时内处理」）

**为什么重要**：举报入口的位置必须对审核员显而易见，深藏在用户主页二级菜单会被认为「无举报机制」。

### 1.3 屏蔽功能

- [ ] 用户可屏蔽/拉黑其他用户（屏蔽后对方内容不再出现，对方无法评论/私信）
- [ ] 屏蔽效果立即生效，无需等待审核
- [ ] 屏蔽列表可管理（可解除屏蔽）

**为什么重要**：1.2 条款明确要求用户有控制自己社区体验的能力。

### 1.4 违规内容处理响应

- [ ] 举报后有实际处理动作（隐藏、删除、警告）
- [ ] 提交时在 Review Notes 中说明处理流程和 SLA
- [ ] 如团队规模小，说明替代机制（如自动过滤 + 延迟 24h 人工）

## 2. UGC 内容审核的最低响应 SLA 建议

Apple 不规定具体 SLA，但 Review Notes 中说明 SLA 可降低拒审概率：

| 内容类型 | 建议 SLA | 说明 |
|---------|---------|------|
| CSAM / 未成年性内容 | 立即（自动过滤） | 必须自动拦截，不依赖人工 |
| 自残/自杀内容 | ≤ 1 小时（人工） | 需有紧急响应机制 |
| 骚扰/仇恨内容 | ≤ 24 小时 | 人工复审 |
| 版权投诉 | ≤ 72 小时 | 需有 DMCA 处理流程（美区） |
| 一般违规举报 | ≤ 48 小时 | 可批量处理 |

## 3. 创作者内容的年龄分级机制

- [ ] 如果平台允许创作者发布成人内容（需要 Apple 额外审批），必须有年龄验证机制
- [ ] 年龄分级问卷中如实填写平台内容等级（不能将含成人内容的平台申报为 4+/9+）
- [ ] 成人内容 App 分级不低于 17+，且需要 Apple 特批（Guideline 1.1.4 色情/成人内容 + 1.2.1(a) 创作者内容年龄验证）
- [ ] 即使在 17+ 分级下，CSAM 永远禁止，无例外

---

## 4. 录制用户活动（相机、麦克风）前的明确同意（5.1.1）

- [ ] 相机使用前：弹出 iOS 系统权限弹窗（必须有合理的 `NSCameraUsageDescription`）
- [ ] 麦克风使用前：同上，`NSMicrophoneUsageDescription` 描述准确
- [ ] 如果录制视频后会上传到服务器：在录制前明确告知上传用途
- [ ] 如果支持直播功能：用户开始直播前必须明确看到「你正在直播，观众可以看到你」的提示
- [ ] 不能静默录制（在用户不知情的情况下开启相机/麦克风）

**为什么重要**：5.1.2 要求数据收集必须在收集时告知目的。隐蔽录制是严重违规（5.1.2(viii)）。

---

## 5. 用户上传 3D 模型 / 设计稿的 UGC 管理要求

适用于允许上传 USDZ、GLB、OBJ、PSD、AI 等文件的创作型社区：

- [ ] 用户上传协议中声明：上传内容不得侵犯第三方知识产权
- [ ] 有 DMCA 投诉/内容下架机制
- [ ] 如果 App 允许用户销售/出售 3D 资产：销售是实物/数字资产需判断是否走 IAP（数字资产交易通常需要走 IAP，详见 `ecommerce.md`）
- [ ] 3D 预览功能不能被包装成「开盲盒」或「随机解锁」的赌博形式（3.1.1 随机虚拟物品披露）

## 6. Review Notes 中说明 Moderation 流程

这是 UGC App 能否顺利通过的关键信息，审核员如果看不到说明会直接以「1.2 违规内容」拒审：

```
【内容审核机制说明】

过滤层：
- 文本：基于 [OpenAI Moderation API / AWS Rekognition / 自研关键词库] 自动过滤
- 图片：基于 [Google Vision SafeSearch / AWS Rekognition] 自动检测
- CSAM：集成 PhotoDNA / NCMEC 哈希匹配，0容忍自动删除

举报流程：
1. 用户长按内容 → 「举报」
2. 选择举报类型 → 提交
3. 系统自动隐藏该内容待审 → 人工审核员在 [24小时] 内处理
4. 违规内容删除 + 发帖用户收到系统通知

屏蔽功能：用户主页右上角「···」→「屏蔽用户」，屏蔽立即生效

审核团队：[X人全职 / 外包团队 / 自动化为主 + 人工复审]
紧急内容（自残/暴力）响应时间：[X小时内]
```

## 常见被拒场景

| 场景 | 被拒条款 | 修复 |
|------|---------|------|
| 无内容举报入口 | 1.2 | 在每条内容旁加举报按钮 |
| 举报后内容仍显示（无处理机制） | 1.2 | 举报后自动隐藏，待人工确认 |
| 用户无法屏蔽骚扰用户 | 1.2 | 在用户主页/聊天界面加屏蔽功能 |
| 发布违规文字未被过滤 | 1.2 | 接入文本审核 API |
| 直播时无「正在直播」提示 | 5.1.2 | 直播开始时全屏提示 + 录制指示灯 |
| 相机使用说明过于模糊 | 5.1.1 | 在 NSCameraUsageDescription 中具体说明用途 |
