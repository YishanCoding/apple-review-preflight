# Games — 游戏类 App 专项

> 本文件只覆盖**游戏类 App 的增量风险**。
> 通用检查项见 `all-apps.md`。

---

## 1. Loot Box（战利品箱）— 概率披露（3.1.1）

**必须在购买前披露**每种物品类型的获取概率。这是硬性要求，不是建议。

- [ ] 每个 Loot Box / 扭蛋 / 卡包的物品概率在**购买确认前**可查
- [ ] 概率按物品稀有度分组展示（例：「传说级 0.3% / 史诗级 2% / 稀有级 15% / 普通级 82.7%」）
- [ ] 概率必须与实际算法一致（Apple 和监管机构可能核查）
- [ ] 概率入口清晰可见（非隐藏在设置→帮助→FAQ 深处）

**为什么重要**：3.1.1 明确要求概率披露。中国区《网络游戏管理办法》也有强制要求。审核员如果发现有 Loot Box 但没有概率说明，直接拒审。

**合规案例**：在 Loot Box 购买界面底部放「查看概率」按钮，点击弹出详细概率列表。

## 2. 禁止使用 IAP 购买真实货币游戏积分（5.3.3）

Apple 明确禁止以下模式：

- 用真实货币（通过 IAP）购买游戏中可当真实货币使用的积分
- 通过游戏内机制将虚拟积分兑换回真实货币
- 用虚拟积分参与赌博并赢取能提现的奖励

**合规边界**：

| 允许 | 不允许 |
|------|-------|
| 用 IAP 购买只能在游戏内消费的金币/钻石 | 用 IAP 购买「游戏币」后可提现 |
| 游戏内购买道具/皮肤 | 游戏内积分可换购实物/现金奖励 |
| 排行榜奖励为游戏内道具 | 竞技比赛奖励为现金（无牌照情况下） |

## 3. 真实货币游戏（Gambling）必须有当地牌照（5.3.1）

- [ ] 如果游戏包含真实货币赌博（老虎机、扑克、体育博彩），必须在 Review Notes 中提交当地合法经营牌照
- [ ] 牌照必须覆盖所有分发地区（每个地区单独牌照）
- [ ] 真实货币赌博 App 仅允许在法律许可的地区展示（使用 Availability 设置限制地区）
- [ ] 不能在没有牌照的情况下以「娱乐赌博」形式规避监管（即使不涉及真实货币提现）

**为什么重要**：5.3.1 是 Apple 的硬限制，没有牌照提交 gambling 类 App 必然被拒，且可能被 Apple 标记账号。

---

## 4. Game Center 规则（4.5.3 / 4.5.5）

- [ ] 不得向第三方披露其他玩家的 Game Center 玩家 ID 或好友关系
- [ ] 不得尝试通过 Game Center 反向查找用户的真实 Apple ID / 邮箱
- [ ] 成就和排行榜数据只能用于游戏内显示，不能用于广告定向
- [ ] 如果集成 Game Center 多人游戏，必须在 App Privacy 中准确声明「游戏中心数据」的使用情况

## 5. 赌博机制判定边界（随机奖励 vs 赌博）

Apple 对「随机奖励」和「赌博」的判定是模糊地带，以下是实践中的判断标准：

### 被认为是「游戏随机奖励」（通常可通过）：
- Loot Box：用游戏内货币（无法提现）购买，获得游戏道具
- 每日登录随机奖励
- PVP 胜利后随机掉落装备
- 扭蛋机（无提现机制）

### 被认为是「赌博」（需牌照或被拒）：
- 老虎机：用 IAP 购买筹码，赢取可提现奖励
- 体育博彩：对真实赛事押注
- 社交赌场（Social Casino）：用真实货币购买游戏币，即使不提现，也在审核灰区
- 彩票/刮刮卡：参与获得真实奖励（现金/礼品卡）

### 灰区判断原则：
- **核心问题**：用户是否能用真实货币「赌」出能兑换真实价值的奖励？
- 如果是→需牌照；如果否→通常被认为是游戏随机奖励

## 6. 免费游戏中消耗品 IAP 的最佳实践

- [ ] 消耗品（金币、体力、道具）在 App Store Connect 中类型设为「Consumable」
- [ ] 消耗品购买后立即到账（不要有延迟或「待发货」状态）
- [ ] 消耗品不提供「恢复购买」（这是正确行为，不是 bug）
- [ ] 虚拟货币面值清晰：「¥30 = 300 金币」，不用模糊的「充值礼包 A/B/C」
- [ ] 最大单次 IAP 金额限制：消耗品单品超过 ¥648（$99.99）需要额外说明用途
- [ ] 不能设计让用户在不知情情况下连续重复购买的流程（dark patterns）

**为什么重要**：消耗品 IAP 是监管和 Apple 双重关注的领域。未成年人保护和防止过度消费是审核重点。

## 7. 游戏抽奖 / 竞技比赛的正式规则要求（5.3.1 / 5.3.2）

### 5.3.1 — 有抽奖/竞赛功能时：

- [ ] 抽奖/竞赛必须有公开的正式规则（Terms & Conditions）
- [ ] 规则需说明：参与资格、奖品描述、中奖概率、开奖时间、奖品发放方式
- [ ] 竞赛奖品为游戏内道具时，列出道具价值（即使是虚拟价值）

### 5.3.2 — 赛事/体育预测类：

- [ ] 如果只有游戏内奖励（不涉及真实货币）：仅需明确规则
- [ ] 如果涉及真实货币奖励：需要当地合法牌照（同 5.3.1 gambling 要求）

---

## 8. 未成年人保护（适用于有 IAP 的所有游戏）

- [ ] 不得专门针对儿童用户设计诱导消费的 IAP 流程
- [ ] 如果年龄分级为 4+ 或 9+，所有 IAP 应有家长控制说明
- [ ] 不得在儿童游戏中直接嵌入广告（Kids Category 完全禁止第三方广告）

## 常见被拒场景

| 场景 | 被拒条款 | 修复 |
|------|---------|------|
| Loot Box 无概率披露 | 3.1.1 | 购买界面加「查看概率」入口 |
| 游戏内虚拟货币可提现 | 5.3.3 | 移除提现功能，或申请 gambling 牌照 |
| 社交赌场 App 无牌照 | 5.3.1 | 提交当地合规牌照 |
| 竞技比赛缺少正式规则 | 5.3.2 | 在 App 内加「比赛规则」页面 |
| 消耗品购买后到账延迟 | 3.1.1 | 购买回调后立即到账，最大延迟 ≤ 3秒 |
