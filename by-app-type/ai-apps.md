# AI Apps — AI/LLM 类 App 专项

> 本文件只覆盖**AI/LLM 功能 App 的增量风险**。
> 通用检查项见 `../by-app-type/all-apps.md`。
> 中国区元数据限制见 `../market-overrides/china.md`。

---

## 1. 禁止误导性 AI 能力声明（5.1.1 / 2.3.1）

- [ ] 不声称 AI 具有专业执照：不说「AI 医生」「AI 律师」「AI 心理咨询师」
- [ ] 不声称 AI 能力超出实际：不说「100% 准确」「和真人专家一样」
- [ ] 必须在功能介绍中说明 AI 输出的局限性和建议人工复核
- [ ] 健康/医疗类建议必须加免责声明：「本功能不构成医疗建议，请咨询专业医生」

**为什么重要**：Apple 正在加强对 AI 应用的 5.1.1 审查。声称医疗/法律专业能力属于误导用户，同时可能触发 5.2.1（不得从事超出 App 功能的业务）。

---

## 2. 用户数据共享给第三方 AI 服务必须明确告知（5.1.1 / 5.1.2）

- [ ] 隐私政策中明确列出第三方 AI 服务商名称（如 OpenAI、Google、Anthropic、Cohere）
- [ ] 说明用户输入内容（prompt）是否发送至第三方服务器
- [ ] 说明第三方是否用用户数据训练模型（如有，需显式同意）
- [ ] App Privacy 数据标签中准确填写「User Content」的收集和共享情况
- [ ] 如果用户可选择「不共享数据」，该设置必须真实生效

**为什么重要**：5.1.2 要求数据收集目的和共享对象透明。AI 类 App 把用户输入发给 OpenAI 等第三方，但隐私政策不提，是近期频繁被拒的新型场景。

---

## 3. AI 生成内容必须有内容审核机制（1.1 / 1.2 / 5.1）

- [ ] 有输入端过滤：防止用户构造 prompt 获取违规内容（越狱 prompt）
- [ ] 有输出端过滤：AI 生成的文本/图片在展示前过一遍内容审核
- [ ] 图像生成功能：过滤 CSAM、裸露、暴力、仇恨内容
- [ ] 文本生成功能：过滤自残/自杀指南、制毒/制武器指南
- [ ] 有用户举报机制（对 AI 生成的不良内容）

**为什么重要**：AI 生成有害内容违反 1.1（令人反感的内容）、1.2（用户生成内容）和 5.1（隐私/安全）。审核员会尝试特定 prompt 测试是否能绕过过滤。没有过滤直接拒审。

---

## 4. 中国区元数据限制（⚠️ 经验规则，非官方条款）

以下品牌词在中国区 App Store 元数据中容易导致被拒（基于 2.3.7 推断 + 实践反馈）：

**高风险词（建议完全避开）**：
- `ChatGPT`、`GPT`、`GPT-4`、`GPT-4o`
- `OpenAI`
- `Claude`、`Anthropic`
- `Gemini`、`Bard`
- `Midjourney`
- `DALL-E`
- `Stable Diffusion`（部分情况）

**涵盖范围**：App Name、Subtitle、Keywords、Description、What's New、截图文字

**合规替代写法**：
- 「基于先进 AI 技术」而非「基于 ChatGPT」
- 「大语言模型驱动」而非「GPT 驱动」
- 「AI 绘图」而非「Midjourney 风格」

**为什么重要**：这是经验性规则（Apple 未公开说明），但多家出海 AI 应用已因此被拒。风险在于：即使通过审核，后续更新时可能被抓。

---

## 5. 用户上传图片前的版权/肖像提示要求（5.1.1 / 5.2.1）

- [ ] 用户上传他人照片（人像生成、换脸、AI 编辑他人图片）前：显示提示「请确认您有权使用此图片」
- [ ] 如果功能涉及公众人物图片生成，需额外说明禁止用于误导性内容
- [ ] 不能提供生成名人虚假图片、Deepfake 裸露图的功能（5.2.1 / 不得伤害他人）
- [ ] 用户协议中声明禁止上传无权使用的图片

**为什么重要**：Apple 对 Deepfake 和非授权人像生成极度敏感，此类 App 经常进入人工复审。

---

## 6. AI 功能在审核期间必须全部可访问（2.1 / 2.1b）

- [ ] API Key 或后端服务在审核期间有效且不限速
- [ ] 如果 AI 功能需要订阅解锁，演示账号必须已订阅（或对演示账号白名单）
- [ ] 不能在审核期间临时关闭「敏感」的 AI 功能（审核员发现功能被隐藏是重大拒审风险）
- [ ] API 接口的地理封锁不包含 Apple 审核 IP（美国旧金山区）

**为什么重要**：审核员无法访问 AI 核心功能时，会以 2.1(a) 拒审。一些开发者错误地以为关掉敏感功能可以规避风险，结果因为功能不完整被拒。

---

## 7. 4.7 条款：迷你 App / 聊天机器人 / 插件的内容过滤要求

条款 4.7 涵盖「迷你 App、迷你游戏、流媒体游戏、聊天机器人、插件和游戏模拟器」，对 AI 聊天机器人有以下明确要求：

- [ ] AI 聊天功能不能扮演「无限制模式」（DAN、越狱人格）
- [ ] 用户设定的 AI 人格/角色不能绕过内容过滤
- [ ] 含有成人内容选项的聊天 App 必须年龄分级为 17+，且需要额外审查
- [ ] 陪伴类 AI（情感陪伴、虚拟恋人）必须在界面上明确标注「AI 非真人」
- [ ] 如果 App 允许用户分发或运行第三方 AI 模型/插件，开发者对所有此类软件负责（4.7）

## 8. Review Notes 必须说明的 AI 专项内容

```
【AI 服务说明】
第三方 AI 服务：[如：OpenAI GPT-4o / Anthropic Claude / Google Gemini]
用途：[用户输入发送至 API，生成回复后展示给用户，不用于模型训练]
数据存储：[不存储 / 存储X天用于改善服务 / 存储用于对话历史]

【内容审核机制】
输入过滤：[使用 OpenAI Moderation API / 自研关键词过滤]
输出过滤：[使用 [服务名] 内容审核 / 自研过滤器]
用户举报：[App 内有「举报内容」功能，入口位置：消息长按菜单]

【AI 功能访问】
演示账号已解锁所有 AI 功能，无使用次数限制。
```

## 常见被拒场景

| 场景 | 被拒条款 | 修复 |
|------|---------|------|
| 用户问「如何制作炸弹」AI 回答了 | 1.1 / 1.2 | 加输入/输出过滤 |
| 元数据含「ChatGPT」（中国区） | 2.3.7 | 替换为「AI」通用描述 |
| AI 图像生成无版权提示 | 5.2.1 | 上传前加确认弹窗 |
| 隐私政策不提 OpenAI | 5.1.1 | 在政策中明确第三方服务商 |
| 审核期间 AI 接口触发限速 | 2.1(a) | 演示账号加白名单或提高限速 |
